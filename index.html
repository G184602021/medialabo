「Webコンテンツ作成」の成果

本テーマの学習目的

HTML, CSS, JavaScript の基本を学び，これらを組み合わせて Web ページを構成する方法を学ぶ。TMLではWeb ページの構造を記述する基本的な方法を理解する。CSS では Web ページの見た目を設定する基本的な方法を理解する。JavaScript では Web ページのインタラクティブな処理を記述する基本的な方法を理解する。

プロジェクト課題について

プロジェクト課題では世界の天気、番組表、グルメ情報の中から好きなものを選び、webページを作成する。HTML,CSS,JavaScriptにプラスしてDOMを使用することでボタンを配置したり、選択肢を表示したり、データから数値を取得したり動きのあるWebページを作成する事ができます。それを利用してここで表示するデータは世界の天気です。ラジオボタンで選択された首都の天気、平均気温、湿度をデータから読み込んで表示する。

工夫点、苦心点、感想

工夫点

今回工夫したことは見やすいをコンセプトにWebページを作成しました。文字や背景色にそれぞれ多くの色を使うと色がゴチャゴチャし見にくくなると思ったので使う色の種類を少なくし、シンプルな見た目にしました。毎回の講義ごとや学習テーマなどを枠でかこむことで見やすくしました。また課題、練習もそれぞれ透明な枠でかこみむことで外の枠の線と二重にならず分けることを可能にし、見やすくしました。プロジェクト課題ではラジオボタンを実装した際に横並びになって見にくかったので縦ならびにすることで見やすくしました。また背景色をグラデーションを使用し天気をイメージする色にしました。天気のデータを表示する際にtableクラスを使用うすることで綺麗にデータ表示する事ができました。

苦心点

今回苦労したことは数当てのプロジェクトでコンソールに表示した内容をページに表示させることです。まずはHTMLの表示させたい場所にスペースフォルダーとしてspan要素を配置させるとことは理解できたのできたのですがどこを検索し、どこに、どのように入れたい内容を検索すれば良いのか理解するのに苦労しました。実際はspan要素を検索しそこに入れたい数値を入れると言う２行で終わる簡単な操作だったのですがDOMに操作が複雑で混乱していてできた時はこんな簡単な操作だったのかと思いました。他にはプロジェクト課題で天気のデータ取得するのにデータの項目名を入れても取得できず苦労しました。先輩に原因を聞くと項目名が配列になっていたからでした。またWebページを利用者にどうしたら見やすくなるのか、利用しやすくなるか考えるのにも苦労しました。あまり美術的な分野が得意ではないので個人的的には全体のレイアウトを考えるのが難しかったです。

感想

今回Webコンテンツ作成通して普段目にしているWebページがすごいんだなと思いました。普段何気なくWebを利用してしましたが今回の学習してこのページではこれが使われているななど分かるようになりました。今回、私が作ったのは簡単なページでしたが苦労しました。それなのに普段見ているページはどんなに作り込まれているんだろうと思いました。他にもWebページはただ実装したいものだけを実装すればいいだけでなく利用者に見やすいように利用しやすいように色使いやレイアウトを考えなければならず、そう言うところも学ぶ必要があるのではないかと思いました。
